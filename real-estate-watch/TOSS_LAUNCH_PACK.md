# Real Estate Watch Toss Launch Pack

Last updated: 2026-03-21

## 1) 토스 제출용 한줄 소개

- 서울·경기 아파트 실거래를 최신 스냅샷과 상위 거래 카드로 빠르게 보는 부동산 정보 서비스

## 2) 토스 제출용 짧은 소개문

- 국토교통부 실거래가 공개시스템과 공공데이터포털 자료를 바탕으로 서울·경기 아파트 매매 실거래를 정리해 보여주는 정보형 서비스입니다.

## 3) 토스 제출용 상세 소개문

- `서울·경기 아파트 실거래`는 서울과 경기권 아파트 매매 데이터를 정적 스냅샷으로 묶어, 가장 비싼 거래 TOP 10과 계약일 최신 거래 TOP 10을 한 화면에서 빠르게 확인할 수 있게 만든 정보형 서비스입니다.
- 실거래 데이터는 국토교통부 실거래가 공개시스템과 공공데이터포털 OpenAPI를 기준으로 정리하며, 앱 안에서는 로딩 부담을 줄인 경량 스냅샷을 우선 제공합니다.
- 지역 필터로 서울, 경기, 전체를 나눠 볼 수 있고, 평균 거래금액, 최고 거래, 최근 계약일 같은 핵심 지표를 카드 형태로 함께 제공합니다.
- 투자 권유나 중개 기능 없이 공개 데이터를 보기 쉽게 정리하는 데 집중한 서비스입니다.

## 4) 스토어/심사 포인트 정리

- 장르: 부동산 정보 / 데이터 카드 / 생활형 정보 서비스
- 핵심 가치: 공개 실거래 데이터를 빠르게 훑고 지역별 흐름을 비교
- 이용 방식: 진입 후 지역 선택 -> 요약 지표 확인 -> 거래 TOP 카드 탐색
- 주요 화면: 상단 요약, 지역 필터, 최고 거래 목록, 최신 계약 목록
- 세션 길이: 짧게 확인하고 나가기 좋은 스낵형 정보 탐색

## 5) 광고 및 수익화 상태

- 현재 버전은 정보형 서비스 흐름을 해치지 않도록 인라인 스폰서 카드 1개만 적용했습니다.
- 광고 위치는 `가장 비싼 거래` 목록과 `계약일 최신 거래` 목록 사이입니다.
- 광고가 핵심 지표와 첫 번째 거래 리스트를 가리지 않도록, 실제 데이터 확인 이후의 중간 구간에만 배치했습니다.
- 기본 빌드 값은 토스 테스트 광고 그룹 ID `ait-ad-test-native-image-id` 입니다.
- 실제 출시 전에는 운영용 광고 그룹 ID로 교체해 QR 테스트를 다시 확인하는 것이 전제입니다.

## 6) 심사 제출 시 운영 메모

- 세로형 정보 레이아웃 기준입니다.
- 토스 앱에서는 인라인 광고 슬롯이 렌더링되고, 일반 웹 미리보기에서는 동일 위치의 안내 카드만 표시됩니다.
- 데이터 로딩은 `GitHub Raw JSON -> Vercel JSON -> 번들 내부 JSON` 순서로 폴백합니다.
- 이 앱은 정보 제공 목적 서비스이며, 투자 권유나 중개 기능은 포함하지 않습니다.

## 7) 썸네일 문안 제안

### A안

- 메인 타이틀: `서울·경기 아파트 실거래`
- 서브 문구: `최신 계약과 최고 거래를 한 번에`

### B안

- 메인 타이틀: `서울·경기 아파트 실거래`
- 서브 문구: `공개 데이터로 보는 거래 흐름`

### C안

- 메인 타이틀: `서울·경기 아파트 실거래`
- 서브 문구: `서울·경기 실거래 TOP 카드`

## 8) QR 테스트 체크리스트

### 기본 동작

- 앱 진입 후 상단 요약 카드가 정상 로딩되는지 확인
- 서울 / 경기 / 전체 필터 전환이 자연스럽게 동작하는지 확인
- 가장 비싼 거래와 최신 계약일 목록이 각각 10건 기준으로 노출되는지 확인
- 스냅샷 소스 표기가 정상적으로 갱신되는지 확인

### 데이터/링크

- 실거래가 공개시스템 링크가 정상 열리는지 확인
- 공공데이터포털 링크가 정상 열리는지 확인
- GitHub Raw / Vercel / 번들 JSON 폴백이 비정상 없이 동작하는지 확인

### 광고

- 토스 앱 안에서만 인라인 광고 슬롯이 렌더링되는지 확인
- 광고가 상단 요약 카드나 첫 거래 리스트를 가리지 않는지 확인
- 광고 노출 실패 시 안내 카드만 보이고 레이아웃이 깨지지 않는지 확인
- 실제 운영용 광고 그룹 ID 적용 후에도 동일 위치와 간격이 유지되는지 확인

## 9) 출시 직전 최종 체크

1. 운영용 `TOSS_INLINE_AD_GROUP_ID` 적용
2. QR 테스트로 광고 노출 / 노필 / 실패 fallback 재확인
3. 소개 문구와 대표 썸네일 최종 확정
4. 정책 링크와 데이터 출처 링크 재확인
5. 정보 서비스 성격과 광고 위치 설명을 심사 메모에 함께 기재
