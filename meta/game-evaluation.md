# 게임 평가표 (v6)

최종 업데이트: 2026-03-09  
기준 문서: 이 파일을 게임 우선순위/리워크/제거 판단의 단일 기준으로 사용

## 1) 목적
이 문서는 `TapTapCho`를 수익 가능한 웹 게임 플랫폼으로 발전시키기 위한 평가 체계다.  
평가는 게임 단위 점수와 플랫폼 단위 점수를 분리해 관리한다.

## 2) 외부 리뷰 반영 핵심 진단
1. 게임성 편차가 큼 (초반 몰입/루프/성장 부족)
2. 클론 성향이 강해 독창성/IP 안전성이 낮음
3. 수익화 준비(광고/리워드/경제 루프) 미흡
4. 기술 안정성/크로스디바이스 운영 표준 미흡
5. 리텐션/마케팅/유저 유입 장치 부족

## 3) 게임 단위 점수 (GQS: Game Quality Score, 100점)

### 3-1. 가중치
| 카테고리 | 가중치 | 평가 포인트 |
|---|---:|---|
| Hook + Core Loop | 25 | 5초 내 이해/재미, 30초 내 보상, 1판 더 유도 |
| Progression + Replay | 20 | 세션 내/세션 간 성장, 반복 동기, 챌린지 |
| Originality + IP Safety | 15 | 클론 탈피, 변형 강도, 저작권 리스크 |
| Polish (Art/Audio/Juice) | 15 | 타격감, 애니메이션, UI 가독성, 사운드 완성도 |
| Technical Stability | 10 | 버그/프리즈/입력 이슈, 브라우저 호환 |
| Mobile UX | 10 | 터치 정확도, 버튼/텍스트 가독성, 손가림 최소화 |
| Monetization Readiness | 5 | 광고 삽입 가능 구조, 리워드 지점, 경제 밸런스 |

### 3-2. 점수 계산
- 각 카테고리 원점수: 1~10
- 총점 계산식: `Σ((원점수/10) * 가중치)`
- 분류:
  - `85+` 출시 후보 (Launch Candidate)
  - `70~84` 소프트런치 후보 (Soft Launch)
  - `50~69` 리워크 필요 (Rework)
  - `<50` 보류/제거 후보 (Hold/Remove)

## 6) 재평가 프로세스 (필수)
1. 10인 플레이 테스트(게임당 10분)
2. 수집 지표:
- 평균 세션 시간
- 1인당 재시도 횟수
- 5초 내 이탈률
- 스테이지별 이탈 히트맵
- D1 재방문 시뮬레이션
3. 정성 질문:
- 재미(1~10)
- 다시 플레이 의향(예/아니오)
- 신선함 체감(1~10)
- 가장 불편한 지점
4. 반영 규칙:
- 테스트 데이터 없는 점수는 `임시`
- 데이터 반영 점수는 `확정`

## 7) 4주 개선 우선순위
1. 상위 2개(AirStrikerLite, Sunken)만 집중 리워크
2. 나머지 Hold 게임은 기능 추가 중단(치명 버그만 수정)
3. 공통 텔레메트리 도입(세션/이탈/재시도)
4. 광고/리워드 지점 설계(플레이 흐름 방해 최소화)
5. 썸네일/랜딩/공유 동선 정비

## 8) 운영 규칙
- 새 게임 추가 시 반드시 이 문서에 `임시 점수` 등록
- 주요 업데이트 후 `확정 점수`로 재평가
- 점수 50 미만 게임은 메인 노출 우선순위에서 제외
- 점수표와 실제 노출 순서가 불일치하면 점수표를 기준으로 수정
